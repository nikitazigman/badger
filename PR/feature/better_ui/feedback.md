1. There is no reason to keep the main section. We can provide an info view, or something similar, that the user can request when needed, but we do not need to waste UI space on a section that probably no one cares about.
2. There is no reason for the user to press Enter to see the contents of B-trees or pages. We can always request and render the content as soon as the user navigates to the line.
3. We do not need both `f` and `F` to select and deselect filters. The user can simply use a single `f` key to select and deselect by navigating to the selected table with `2`, then pressing `f` to select and pressing `f` again on the selected row to deselect. We could also use Enter for this.
4. We can use `[4]` and `[5]` to let the user navigate to the detail view and meta info view, meaning the middle and rightmost views.
5. We need a better way to represent the page data. I am not sure the middle and right panels are the best way to do this. Maybe we can introduce several views of the data instead. For example, we could have a raw hex view where all data segments are isolated by colors or frames, and the user can use the arrow keys to jump between data blocks. I do not know the exact solution, but we need to improve this. It is not very clear when I open it.
